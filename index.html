02 / agents

Агенты, которые помогают строить проект

VS Code делает AI-агентов частью рабочего процесса: агент может планировать, менять код, запускать команды, итеративно исправлять ошибки и работать параллельно с ручной разработкой.

Agent mode: задача → план → изменения → проверка

Агент может взять задачу, предложить план, внести изменения в файлы, запустить команды и продолжать итерации до готового результата, пока разработчик контролирует итог.

prompt.md
# Задача для agent mode
Исправь ошибку расчёта risk score:
- добавь unit-тесты для ночных транзакций;
- вынеси веса модели в конфигурацию;
- запусти тесты;
- покажи diff перед применением.

Все AI-сессии в одном представлении

На главной странице VS Code подчёркивается идея параллельной работы нескольких агентов: можно отслеживать, фильтровать и открывать отдельные agent-сессии без переключения в другой инструмент.

agent-sessions.json
{
  "sessions": [
    { "task": "fix-tests", "status": "running", "agent": "local" },
    { "task": "homepage-redesign", "status": "pull-request", "agent": "cloud" }

}

Custom instructions и правила команды

VS Code поддерживает workspace-инструкции, например .github/copilot-instructions.md, чтобы ответы Copilot соответствовали стеку, стилю кода, требованиям безопасности и документации.

.github/copilot-instructions.md
# Copilot instructions
- Пиши код на TypeScript strict mode.
- Для Python используй type hints и pytest.
- Не добавляй зависимости без объяснения.
- Для API возвращай понятные ошибки и HTTP status codes.
- Перед изменениями в нескольких файлах сначала предложи план.

Расширения, MCP и внешние инструменты

VS Code позволяет расширять возможности агентов через расширения, Model Context Protocol servers и инструменты команды, чтобы AI работал с реальным контекстом проекта.

04 / code cells

Полноценные кодовые ячейки для демонстрации в VS Code

Ниже — корректные, самодостаточные примеры для Python, JavaScript, JSON и workspace-инструкций. Каждую ячейку можно скопировать.

analyze_logs.py
from collections import Counter

events = [
    {"user": "alice", "event": "login", "status": "ok"},
    {"user": "bob", "event": "login", "status": "failed"},
    {"user": "bob", "event": "login", "status": "failed"},
    {"user": "carol", "event": "download", "status": "ok"},
]

failed_by_user = Counter(
    item["user"]
    for item in events
    if item["status"] == "failed"
)

for user, count in failed_by_user.most_common():
    print(f"{user}: {count} failed login(s)")

Демонстрация: попросить Copilot объяснить генератор, добавить чтение CSV или создать pytest для подсчёта failed login.

risk.js
function scoreRisk(transaction) {
  const amountScore = Math.min(transaction.amount / 10000, 1) * 0.65;
  const geoScore = transaction.country === "RU" ? 0 : 0.2;
  const timeScore = transaction.hour <= 5 ? 0.15 : 0;
  return Number((amountScore + geoScore + timeScore).toFixed(3));
}

const tx = { amount: 8700, country: "TR", hour: 2 };
console.log({ risk: scoreRisk(tx), decision: scoreRisk(tx) >= 0.55 ? "review" : "approve" });

.github/copilot-instructions.md
# Project coding rules

Use these rules for every Copilot response:
- Python code must include type hints and small pure functions.
- JavaScript code must avoid implicit globals and use const/let.
- Tests are required for new business logic.
- Explain security assumptions before suggesting authentication code.
- Prefer readable code over clever one-liners.

Демонстрация: этот файл можно хранить в репозитории, чтобы команда получала более согласованные AI-ответы.

05 / workflow

Практический сценарий VS Code + AI

В учебной или профессиональной работе VS Code можно показать как единый контур: код, чат, агент, терминал, тесты, Git и расширения.

1

Открыть проект и включить AI-функции

Через Copilot icon в Status Bar пользователь включает AI Features, выполняет вход и получает доступ к chat/interactions в рамках своего плана.

2

Задать правила проекта

Создайте .github/copilot-instructions.md, чтобы AI учитывал стек, стиль, правила безопасности, тестирование и документацию.

3

Поручить агенту ограниченную задачу

Например: «добавь тесты для функции risk score», «объясни failing test», «сделай рефакторинг без изменения API».

4

Проверить diff, тесты и Git-историю

AI ускоряет работу, но ответственность остаётся у разработчика: проверить diff, запустить тесты, оценить безопасность и зафиксировать изменения.
